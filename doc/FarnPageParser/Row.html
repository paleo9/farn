    class FarnPageParser::Row

    class Row Parse and store one row from the farn results table. Relevant sections begin with a tr column_name_ data n, e.g. manuf_data_04. The headings for properties is stored in the containing FarnPageParser, not in the Row.

    Attributes

    brand_name[R]

    Manufacturer’s brand name.

    description[R]

    Description of this part, e.g. ‘resistor, carbon comp’.

    manufacturer_part_no[R]

    Manufacturer’s part number.

    prices[R]

    An array of QPrice to store the price-per-quantity for this part.

    properties[R]

    Extended properties for this part, e.g. resisitance, power rating.

    supplier_part_no[R]

    Farn’s order code for this part.

    weblink[R]

    Link to Farn’s page for this part.

def to_hash(prop_headings)
  h = {
    manufacturer_part_no: @manufacturer_part_no,
    supplier_part_no: @supplier_part_no,
    brand_name: @brand_name,
    description: @description,
    weblink: @weblink
  }
  h[:properties] = properties_to_hash_array(prop_headings)
  h[:prices] = prices_to_hash_array
  h
end
    to_s() click to toggle source

    Returns a string of all the information for this row. It does not include the property_headings, which are found in the containing FarnPageParser as this would result in a need to pass an argument to #to_s, possibly go against convention.

    # File farn_page_parser.rb, line 160
def to_s
  s = "Supplier part no: #{@supplier_part_no}" +
  "Manufacturer part no: #{@manufacturer_part_no}" +
  "Brand name: #{@brand_name}" +
  "Description: #{@description}" +
  "Suppliers web page: #{@weblink}" +
  'Properties: ' + @properties.join('; ') +
  'Prices: ' + @prices.join('; ')
  s
end
